EmathGeoeditor

Create geometric constructions

EmathGeoeditor depends on jQuery, JSXGraph. EmathGeoeditor is otherwise a selfcontained js-file containing all css and images in one file to make installation easier. Additionally you can use provided colpicker.js to get fancier color picker instead of the default rgb(a)-hexacodes.

Example 1

Here is an example, how to use gedit.js.

<div id="box1"> </div> <script type="text/javascript"> jQuery('#box1').geditor({editable: true}); </script>

The geoeditor is started in edit mode. You can click the checkbox to toggle between edit mode and show mode.

Example 2

Here is another example.

<div id="box2"> </div> <script type="text/javascript"> var data2 = ... (see the variable data2 in the source code of this html page) ...; data.editable = true; jQuery('#box2').geditor(data2); </script>

The geoeditor is started in edit mode with existing data.

Example 3

Here is yet another example.

<div id="box3"> </div> <script type="text/javascript"> var data2 = ... (see the variable data2 in the source code of this html page) ...; data.editable = false; jQuery('#box3').geditor(data2); </script>

The geoeditor is started in show mode with existing data.

Example 4

Here is yet another example.

<div id="box4"> </div> <script type="text/javascript"> var data2 = ... (see the variable data2 in the source code of this html page) ...; data.editable = false; data.browsingMode = 'tabless'; jQuery('#box4').geditor(data2); </script>

The geoeditor is started in show mode with existing data and with navigation dots instead of tabs.

Usage

The size of the editor or drawing is determined by the containing element. By default the width will be 100% of the containing element and the height will be computed from the aspect ratio of the drawing.

jQuery plugin

The edit or show mode is initiated in any html-element with .geditor()-method of jQuery-object containing that html-element. The data of drawing(s) can be given as an object argument. The data of the existing drawing can be requested from it with .geditor('getdata').

In edit mode:

  • Layout
    • Toolbar at the top (Some tools have "subtools".)
    • Drawing area in the middle
    • Tabs and handling of tabs below drawing area.
    • Info area with help for current tool in the bottom.
    • Objectlist at right hand side with eye icon for "show/hide" and trahscan for removing.
    • Propertyarea at top right hand side showing editable properties of current object. Basic properties are shown by default and advanced properties come available by clicking three dots (...) in the bottom of propertyarea.
  • Tabs
    • One editor can contain several drawings.
    • Each drawing is in its own tab.
    • Tabs can be used, for example, to illustrate steps of a construction.
    • Tabs can be added, removed and copied.
    • The first object in objectlist is the drawing in current tab itself.
    • Properties of a drawing include: name (label of the tab), description (caption of the drawing), dimensions of the drawing, grid on/off, axis on/off, static (lock the drawing to non-interactive state in show mode) and allow controls (show navigation and zooming in show mode).
  • Drawing objects
    • There are several different kind of geometrical objects that can be added. For example: point, line, circle, glider (point on some curve), triangle, rectangle, normal, parallel line, etc.
    • Each object in current tab/drawing is listed in object list.
    • Some objects depend on other objects (for example, line depends on two points) and will be updated, when those are moved.
    • There are three kinds of points: Solid points are free points that can be moved freely, hollow points (white inside) are restricted points that can be moved with some restrictions (for example, glide on a line), diamonds and plusses are fully dependent points that can not be moved (fourth point of rectangle, intersections).
    • If an object is removed and another, remaining, object depended on the removed one, the remaining one will not be drawn and it will be marked with red color in the list to indicate it is broken.
    • Broken objects can either be removed or repaired by fixing their dependencies in their property settings.
    • If colpicker.js is used, the colorbars in propertyarea are clickable and give a small palet for selecting a color.
  • Localization
    • The geoeditor has localization system to show texts in different languages.
    • The editor finds the closest parent in DOM-tree that has lang-attribute set and uses that language, if it is available. English is the default language. (So, if for example the body-element has lang="fi", the editor will use Finnish language.
    • Available languages so far: English, Finnish, Swedish (partial)

In show mode:

  • Layout
    • Drawing area
    • Tabs or navigation dots under the drawing area
    • Caption of the drawing in the current tab under the tabs.
  • Tabs
    • Separate drawings can be selected either with tabs or with dot navitation.
    • The tabsystem is selected with browsingMode-property in the drawing data. (No user interface for selecting this yet.)
    • If tabs are used, the label of the tab is the name of the drawing or a number, if the name is empty.
    • Each drawing can have navigation (browsing the view left, right, up, down and zooming) enabled or disabled. This is selected in the edit mode or in the raw data with allowControl-property.
